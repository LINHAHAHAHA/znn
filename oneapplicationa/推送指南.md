# 🚀 快速推送到GitHub并自动构建APK

## 最简单的方法

**双击运行**：`推送到GitHub.bat`

这个脚本会：
1. ✅ 添加所有必要的文件（包括gradlew）
2. ✅ 提交到Git
3. ✅ 推送到GitHub
4. ✅ 自动触发构建

## 手动推送（如果脚本不工作）

在项目目录打开PowerShell或命令行：

```bash
# 1. 强制添加gradlew文件
git add -f gradlew
git add -f gradlew.bat
git add -f gradle/wrapper/gradle-wrapper.jar
git add -f gradle/wrapper/gradle-wrapper.properties

# 2. 添加其他文件
git add .

# 3. 提交
git commit -m "完整的Android项目文件"

# 4. 推送
git push
```

## 推送后

1. **打开GitHub仓库**
2. **点击 Actions 标签**
3. **查看构建进度**（约5-10分钟）
4. **下载APK**：
   - 构建完成后
   - 在Actions页面底部找到 **Artifacts**
   - 下载 **公安接处警平台-debug.zip**
   - 解压得到APK文件

## 验证文件是否在GitHub上

推送后，在GitHub仓库页面检查这些文件是否存在：
- ✅ `gradlew`
- ✅ `gradlew.bat`
- ✅ `gradle/wrapper/gradle-wrapper.jar`
- ✅ `gradle/wrapper/gradle-wrapper.properties`
- ✅ `.github/workflows/build-apk.yml`

如果这些文件都在，GitHub Actions就能成功构建APK了！

## 常见问题

### Q: 推送失败，提示没有权限？
A: 需要先配置Git凭据或使用Personal Access Token

### Q: 推送成功但构建失败？
A: 查看Actions页面的构建日志，找到具体错误信息

### Q: 构建成功但找不到APK？
A: 在Actions页面底部的Artifacts区域下载

## 🎉 成功标志

当你看到：
- ✅ GitHub Actions显示绿色对勾
- ✅ Artifacts区域有可下载的文件
- ✅ 下载的zip包含app-debug.apk

恭喜！你的APK构建成功了！
